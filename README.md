# 🚀 Система управления банковскими картами

Backend-приложение на Java Spring Boot для управления пользователями, банковскими картами и переводами между своими картами.

Проект реализует REST API с аутентификацией через JWT, ролевым доступом, защитой чувствительных данных, миграциями базы данных через Liquibase и документацией API через Swagger/OpenAPI.
## 📝 Описание задачи

Приложение предназначено для управления банковскими картами и включает:

- создание и управление картами;
- просмотр карт;
- переводы между своими картами;
- разграничение доступа по ролям;
- защиту чувствительных данных;
- документированное REST API.

## 💳 Атрибуты карты

Каждая банковская карта содержит:

- зашифрованный номер карты;
- маскированное отображение номера в формате `**** **** **** 1234`;
- владельца карты;
- срок действия;
- статус карты:
    - `ACTIVE` — активна;
    - `BLOCKED` — заблокирована;
    - `EXPIRED` — истёк срок действия;
- баланс;
- признак запроса на блокировку;
- даты создания, обновления и логического удаления.

## 🔐 Аутентификация и авторизация

В проекте используется Spring Security и JWT.

Реализованы роли:

- `ADMIN`
- `USER`

После успешной авторизации пользователь получает JWT access token и передаёт его в заголовке `Authorization` в формате `Bearer <access_token>`.

Доступ к административным endpoint'ам разрешён только пользователям с ролью `ADMIN`.

## ✅ Возможности администратора

Администратор может:

- создавать пользователей;
- просматривать пользователей;
- обновлять данные пользователей;
- блокировать и активировать пользователей;
- логически удалять пользователей;
- создавать банковские карты;
- просматривать все карты;
- просматривать заявки пользователей на блокировку карт;
- блокировать карты;
- активировать карты;
- логически удалять карты.
## ✅ Возможности пользователя

Пользователь может:

- просматривать свои карты;
- смотреть баланс своей карты;
- запрашивать блокировку своей карты;
- выполнять переводы между своими картами;
- просматривать историю своих переводов.

Пользователь не может получать доступ к чужим картам и переводам.

## 🧾 API

В проекте реализованы REST endpoint'ы для:

- аутентификации;
- управления пользователями;
- управления картами;
- просмотра пользовательских карт;
- запроса блокировки карты;
- переводов между своими картами.
### Auth

`POST /api/auth/login` — авторизация пользователя и получение JWT access token.

### Admin Users

`GET /api/admin/users` — получить список пользователей.

`GET /api/admin/users/{id}` — получить пользователя по идентификатору.

`POST /api/admin/users` — создать пользователя.

`PATCH /api/admin/users/{id}` — обновить данные пользователя.

`PATCH /api/admin/users/{id}/role` — изменить роль пользователя.

`PATCH /api/admin/users/{id}/block` — заблокировать пользователя.

`PATCH /api/admin/users/{id}/activate` — активировать пользователя.

`DELETE /api/admin/users/{id}` — логически удалить пользователя.

### Admin Cards

`GET /api/admin/cards` — получить список карт.

`GET /api/admin/cards/{id}` — получить карту по идентификатору.

`GET /api/admin/cards/block-requests` — получить список заявок на блокировку карт.

`POST /api/admin/cards` — создать банковскую карту.

`PATCH /api/admin/cards/{id}/block` — заблокировать карту.

`PATCH /api/admin/cards/{id}/activate` — активировать карту.

`DELETE /api/admin/cards/{id}` — логически удалить карту.

### User Cards

`GET /api/cards/my` — получить список своих карт.

`GET /api/cards/my/{id}` — получить свою карту по идентификатору.

`GET /api/cards/my/{id}/balance` — получить баланс своей карты.

`PATCH /api/cards/my/{id}/block-request` — запросить блокировку своей карты.

### User Transfers

`POST /api/transfers` — выполнить перевод между своими картами.

`GET /api/transfers/my` — получить историю своих переводов.

`GET /api/transfers/my/{id}` — получить свой перевод по идентификатору.

## 🔒 Безопасность

В проекте реализованы следующие меры безопасности:

- пароли пользователей хранятся в виде BCrypt-хэшей;
- полный номер карты не возвращается через API;
- номер карты хранится в базе данных в зашифрованном виде;
- для отображения используется только маска номера карты;
- доступ к endpoint'ам ограничен ролями `ADMIN` и `USER`;
- пользовательские операции выполняются только над ресурсами текущего авторизованного пользователя;
- JWT используется для stateless-аутентификации.
## 🗄️ Работа с базой данных

В качестве базы данных используется PostgreSQL.

Миграции выполняются через Liquibase.

Путь к миграциям: `src/main/resources/db/migration`.

Основные таблицы проекта:

- `users`;
- `cards`;
- `transfers`.

Также Liquibase создаёт служебные таблицы:

- `databasechangelog`;
- `databasechangeloglock`.

Перед запуском необходимо поднять базу данных PostgreSQL и указать настройки подключения в `application.yml`.

`docker compose up -d`

Запуск приложения:

`./mvnw spring-boot:run`

После запуска приложение доступно по адресу:

`http://localhost:8080`
